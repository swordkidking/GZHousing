# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 项目背景

本仓库是**广州安居集团数字化中台实施项目**的文档管理目录，覆盖项目全生命周期各阶段的文档，包括调研、需求、设计、开发、测试、部署、验收及运维。

## 目录结构与用途

```
01-调研/          业务现状调研报告、访谈记录、现有系统梳理
02-需求/          需求规格说明书、用户故事、需求变更记录
03-设计/
  ├─ 架构设计/    系统架构图、技术选型说明
  ├─ 数据模型/    ER图、数据字典、表结构设计
  ├─ 接口设计/    API规范、接口联调文档
  └─ UI设计/      原型图、交互说明
04-开发/
  ├─ 技术方案/    各模块技术实现方案
  └─ 开发规范/    编码规范、Git规范、Review规范
05-测试/          测试计划、测试用例、测试报告、缺陷记录
06-部署/
  ├─ 环境配置/    各环境（开发/UAT/生产）配置说明
  └─ 操作手册/    部署步骤、回滚方案
07-验收/          验收标准、验收报告、甲方确认函
08-运维/
  ├─ 运维手册/    日常巡检、备份策略、监控告警
  └─ 故障记录/    线上问题记录与复盘
09-会议纪要/      各阶段评审会、周例会纪要
10-项目管理/
  ├─ 计划与进度/  项目计划、里程碑、周报
  ├─ 变更记录/    需求变更、范围变更的审批记录
  └─ 问题与风险/  问题清单、风险登记册
```

## 文档命名规范

- 格式：`YYYYMMDD_文档类型_主题.扩展名`
- 示例：`20260415_需求规格_房源管理模块_v1.0.docx`
- 版本号：重要文档需带版本号后缀（`_v1.0`、`_v1.1`）
- 正式交付文档优先使用 `.md` 或 `.pdf`，过程文档可使用 `.docx`

## 文档写作要点

- 调研文档：需注明调研时间、访谈对象、现状问题和改进建议
- 需求文档：每条需求需有唯一编号（如 `REQ-HOUSE-001`），标注优先级和来源
- 设计文档：架构图和流程图优先使用 Mermaid 语法内嵌在 `.md` 文件中，便于版本管理
- 会议纪要：需包含时间、参会人、决议事项、待办责任人和截止时间
- 变更记录：每次变更需记录变更原因、影响范围和审批人
